## 人工修改内容

虽然 AI 提供了基础代码框架，但进行了以下深度修改和验证：

### 1. 物理约束验证
- **关键发现**：AI 生成的初始环境在燃料耗尽后推力仍然存在，不符合物理
- **人工修正**：添加严格燃料约束，燃料耗尽后推力强制为0
- **验证方式**：手动键盘控制测试每个物理步骤
- **结果**：训练更困难，但物理模型正确

### 2. 奖励函数调优
- AI 生成的初始奖励函数导致火箭在半空悬停耗尽燃料
- 人工调整了高度惩罚和着陆奖励的比例
- 加入了速度惩罚的动态权重（接近地面时更重视速度）
- 增加了动作平滑惩罚，减少油门抖振
- **关键改进**：设计分层速度目标，引导学习最优策略（高空自由落体，低空减速）

### 3. 训练参数优化
- 对比了不同学习率（1e-4, 3e-4, 5e-4）的表现
- 测试了不同网络结构（[128,128] vs [256,256]）
- 调整了评估间隔和早停阈值

### 4. 鲁棒性测试设计
- 设计了5种扰动场景（随机高度、随机速度、重力扰动、传感器噪声等）
- 实现了串级PID控制器作为基线对比
- 验证了模型对环境扰动的抗干扰能力

### 5. 代码质量保证
- 检查了所有脚本的错误处理和边界情况
- 修复了JSON序列化、动作格式兼容性等bug
- 确保所有脚本可以独立运行

## 最终训练结果

### 物理正确环境下的训练成果
- **环境**：严格燃料约束（燃料耗尽后推力为0）
- **训练步数**：500,000 步
- **成功率**：100%（100次测试）
- **着陆速度**：1.156 m/s（< 2 m/s 限制）
- **燃料使用**：4.359 kg（共 5 kg）
- **着陆时间**：7.20 秒

### 学习到的策略
1. **高空阶段**（h > 30m）：自由落体，节省燃料
2. **中空阶段**（15m < h < 30m）：开始观察，准备减速
3. **低空阶段**（h < 15m）：使用推力减速
4. **着陆阶段**（h < 5m）：精确控制，确保软着陆

### 关键技术突破
1. **燃料管理**：智能体学会了合理分配燃料，避免过早耗尽
2. **推力控制**：根据高度和速度动态调整推力
3. **鲁棒性**：在各种初始条件下都能成功着陆

## 注意事项

1. AI 生成的代码需要人工验证和调优
2. 训练结果受随机种子和超参数影响
3. 不同环境下的鲁棒性表现可能不同
4. 奖励函数需要根据具体任务目标调整
5. 物理约束的正确性是训练成功的前提
# Reproducible v2 AI usage note

This project used AI assistance as an engineering aid for environment
inspection, implementation suggestions, script refactoring, and documentation
drafting. The final experimental claims must come from locally executed
training/evaluation commands, especially the JSON/CSV files under
`results/reproducible/...`.

Human verification remains required for:

- installing the Python 3.10/3.11 environment and dependencies;
- running PPO training and the core experiments;
- checking that `eval_episodes.csv` contains randomized initial conditions;
- confirming the final success rate, failure reasons, figures, and report
  tables match the generated result files.
- verifying the pure energy-guided PPO experiment from the official
  from-scratch run
  `results/reproducible/energy_ppo_from_scratch_time/baseline_vs_pure_energy_ppo.json` before
  making any claims about whether the energy reward converged.

Do not describe the project as "fully generated by AI". A suitable statement is:
"AI tools were used to help inspect the existing codebase, propose refactors,
and generate initial versions of scripts and documentation. The experiment
configuration, code review, execution, result verification, and final report
selection were manually checked against the generated JSON/CSV outputs."
